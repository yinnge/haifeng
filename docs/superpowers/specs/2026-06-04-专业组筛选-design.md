# 专业组筛选（两步式）设计

**日期**：2026-06-04
**状态**：草案（待用户 review）
**作者**：brainstorming 会话

---

## 1. 目标

为现有 `GET /api/v1/app/admission/group/page` 端点新增 4 大维度的筛选能力（大学 / 城市 / 专业 / 学科评估），同时支持大学名模糊搜索。与现有的安全系数 Redis ZSet 范围查询能力叠加，实现完整的多条件 + 安全系数组合筛选。

**不涉及**：表结构变更（无 Flyway 迁移）、专业组-专业 关系变更。

---

## 2. 工作流形态（两步式）

```
Step 1: GET /filter/universities
        入参：tags / nature / famousUnion / category / educationLevel / department
              / provinces / regions / populationBucket / evaluationGrades
        出参：[{id, name, cityName, tags}]   ← 用户多选
        ──> 拿回 universityIds

Step 2: GET /filter/majors
        入参：majorCategories / parentCategories / majorTags / majorTypes
        出参：[{id, majorCode, majorName, majorCategory, majorTags}]
        ──> 拿回 majorCodes

Step 3: GET /group/page
        入参：原有 (batch, subjectFilter, minSafetyLevel, maxSafetyLevel, orderBy, page, size)
             + 新增 (universityIds, majorCodes, universityName)
        出参：IPage<AdmissionGroupPageVO>（不变）
```

辅助接口：`GET /filter/tags` 返回大学 tags 字典（前端下拉数据源）。

**过滤语义**：
- 同一子字段多选 → OR
- 不同子字段（跨子字段、跨维度）→ AND
- 维度一（大学）= tags / nature / famousUnion / category / educationLevel / department
- 维度二（城市）= provinces / regions / populationBucket
- 维度三（专业）= majorCategories / parentCategories / majorTags / majorTypes
- 维度四（学科评估）= evaluationGrades

## 4. 详细设计

### 4.1 DTO 字段表

**`AdmissionGroupQueryDTO`（改，加 3 字段）**

```java
@NotBlank private String batch;
private Boolean subjectFilter = false;
private Integer page = 1;
private Integer size = 20;
private BigDecimal minSafetyLevel;
private BigDecimal maxSafetyLevel;
private String orderBy = "safety_desc";

// ===== 新增 =====
private List<Long> universityIds;    // null/空=不限，来自 /filter/universities
private List<String> majorCodes;     // null/空=不限，来自 /filter/majors
```

**`AdmissionUniversityOptionVO`（新）**
```java
private Long id;                    // t_universities.id（雪花主键，前端调用 group/page 时回传）
private String name;                // 大学名
private String cityName;            // 城市名（前端展示用）
private List<String> tags;          // 大学 tags 列表（前端展示用）
```

**`AdmissionMajorOptionVO`（新）**
```java
private Long id;                    // t_major.id（**仅用于前端展示/去重，不参与 group/page 过滤**）
private String majorCode;           // t_admission_major_score.major_code（**前端必须传这个回 group/page**）
private String majorName;
private String majorCategory;
private String majorTags;
```

> **前后端约定**：`/filter/majors` 返回的 `id` 只用于前端展示/去重；前端调用 `group/page` 时使用 `majorCode`（List<String>），**不要传 `id`**。`t_major.id`（雪花主键）与 `t_admission_major_score.major_code`（招生专业代码）是不同体系。

### 4.2 PopulationBucketEnum → SQL 区间

| 枚举值 | 区间 | SQL 条件 |
|--------|------|----------|
| `LT_2000` | `[0, 2000)` | `population >= 0 AND population < 2000` |
| `BTW_2000_3000` | `[2000, 3000)` | `population >= 2000 AND population < 3000` |
| `BTW_3000_4000` | `[3000, 4000)` | `population >= 3000 AND population < 4000` |
| `GTE_4000` | `[4000, +∞)` | `population >= 4000` |

`GTE_4000` 没有上限（业务常识：城市常住人口最多 4000+ 万级别，PG `>=` 即可）。

### 4.3 关键 SQL 片段

**大学筛选（动态拼接）**：

```sql
SELECT DISTINCT u.id, u.name, u.city_name, u.tags
FROM t_universities u
[LEFT JOIN t_city c ON c.city_name = u.city_name]
[INNER JOIN t_subject_evaluation se ON se.university_id = u.id]
WHERE u.status = 1
  [AND u.tags && CAST(#{tags} AS text[])]
  [AND u.nature IN (#{nature})]
  [AND u.famous_union IN (#{famousUnion})]
  [AND u.category IN (#{category})]
  [AND u.education_level IN (#{educationLevel})]
  [AND u.department IN (#{department})]
  [AND u.province_name IN (#{provinces})]
  [AND u.region IN (#{regions})]
  [AND c.resident_population >= #{minPop} AND c.resident_population < #{maxPop}]
  [AND se.evaluation_grade IN (#{evaluationGrades})]
ORDER BY u.sort_order DESC, u.id DESC
LIMIT 500
```

**大学 tags 字典**：

```sql
SELECT DISTINCT unnest(tags) AS tag
FROM t_universities
WHERE status = 1 AND tags IS NOT NULL
ORDER BY tag
```

**专业筛选**：

```sql
SELECT id, major_code, major_name, major_category, major_tags
FROM t_major
WHERE status = 1
  [AND major_category IN (#{majorCategories})]
  [AND parent_category IN (#{parentCategories})]
  [AND major_tags IN (#{majorTags})]
  [AND major_type IN (#{majorTypes})]
ORDER BY id DESC
LIMIT 1000
```

**group/page 的 `majorCodes` 过滤（内存层）**：

```sql
SELECT DISTINCT group_id
FROM t_admission_major_score
WHERE group_id IN (#{groupIds})
  [AND major_code IN (#{majorCodes})]
```

返回的 `group_id` 集合用于在内存层做交集。

### 4.4 group/page 双路径设计

```
入参：(universityIds, majorCodes, universityName) 全部为空？
   │
   ├─ 是 → 走原 ZSet 范围分页（保持现状，零回归）
   │
   └─ 否 → ZSet 全量拉取 + 内存过滤 + 内存分页
              │
              ├─ ZSet 拉：rangeByScoreWithScores(key, min, max)  ← 不分页
              ├─ 内存按 universityIds 过滤（HashSet O(1)）
              ├─ 查 t_admission_major_score 做 majorCodes 过滤（一次 DB）
              ├─ 内存按 universityName ILIKE 过滤
              └─ 内存分页 + selectBatchIds 回查
```

**为什么分两条路径**：
- 无新筛选时，ZSet 范围分页是亚毫秒的；保持现状避免回归
- 有新筛选时，必须先全量 ZSet 拉，再在内存/DB 过滤；预估 groupIds ≤ 几千，Redis 拉一次 OK
- 切换条件简单：`universityIds.isEmpty() && majorCodes.isEmpty() && (universityName == null || universityName.isBlank())`

### 4.5 兜底与降级

| 场景 | 行为 |
|------|------|
| Redis 不可用 + 无新筛选 | 走 `pageGroupsForPaidFallback`（现有） |
| Redis 不可用 + 有新筛选 | `pageGroupsForPaidFallback` 在原有 DB 全量计算路径上加 3 个新参数过滤（纯 SQL 拼接，无 ZSet） |
| 筛选后结果为 0 | 返回空 IPage + `total=0`（**不抛异常**） |
| 免费用户 | `pageGroupsForFree` 加 3 个新参数过滤，**10 条限制 + masked 占位符保持** |

## 9. 性能与索引

所有查询都用现有索引：

- `idx_univ_tags`（GIN）→ tags `&&` 操作走索引
- `idx_univ_province` / `idx_univ_category` / `idx_univ_status`
- `idx_se_university` / `idx_se_grade`
- `idx_city_population`

`/filter/universities` 默认 `LIMIT 500`，超过截断 + 提示"筛选结果过多，请增加筛选条件"。
`/filter/majors` 默认 `LIMIT 1000`。

## 10. 测试要点

**单元测试**：

- [ ] 4 维度单选/多选 SQL 拼接正确
- [ ] 跨子字段 AND 拼接正确
- [ ] tags 用 `&&` 数组操作符
- [ ] 4 个人口桶枚举 → SQL 区间正确
- [ ] `PopulationBucketEnum.GTE_4000` 上限 = `+∞`
- [ ] 大小写不敏感（PG `ILIKE`）
- [ ] DTO 边界校验（universityIds 长度、majorCodes 长度、min/max safety）

**集成测试**：

- [ ] 完整两步式：filter/universities → filter/majors → group/page
- [ ] 付费用户走 ZSet + 新筛选的内存过滤
- [ ] 免费用户走 DB + 10 条限制 + masked 占位符
- [ ] Redis 不可用时降级到 DB 路径
- [ ] **回归：不传新筛选 → 原 ZSet 分页（不能破）**
- [ ] 5 个安全区间档位（[0,0.3) / [0.3,0.5) / [0.5,0.7) / [0.7,0.85) / [0.85,1.0)），对应 搏/冲/稳/保/垫

**边界用例**：

- [ ] 全部维度都选 → 结果应正常分页
- [ ] 维度互斥（如 985 + 专科，可能 0 结果）
- [ ] 模糊搜索不存在的大学名
- [ ] 不存在的 universityId / majorCode → 静默忽略

模块划分方案
模块一：统一搜索（新增 V19）
- t_job_index — 全局岗位聚合索引
- 职责：全站搜索/筛选/排序/推荐入口，所有岗位表通过 Service 层同步写入
- 独立模块，不依赖其他模块的业务逻辑
模块二：体制内招录（拆分自现有，升级为独立模块）
- t_civil_position 🔜 NEW
- t_institution_position 🔜 NEW
- t_military_position 🔜 NEW
- t_selection_position 🔜 NEW
- 共性：国考/省考/事业编/部队文职/选调生，统一考试体系，按 exam_type 区分
模块三：行业专项招聘（新增 V20）
- t_teacher_position 🔜 NEW
- t_healthcare_position 🔜 NEW
- t_finance_position 🔜 NEW
- 共性：行业垂直细分，各自有独立资质体系（教师资格证/执业医师/银行从业），不能混用通用模板
模块四：基层/社区/公益岗（新增 V21）
- t_grassroots_project_position 🔜 NEW
- t_community_position 🔜 NEW
- t_public_welfare_position 🔜 NEW
- 共性：门槛低、政策性岗位（服务期/期满政策/补贴标准），而非市场性招聘
模块五：内容管理（新增 V22）
- t_exam_guide 🔜 NEW（替代 t_civil_methodology + t_institution_guide）
- t_notice 🔜 NEW（替代 t_recruitment_notice + t_institution_notice + t_military_notice）
- 共性：纯内容型（文章+公告），不涉及岗位业务逻辑，独立管理后台
  模块间关系
  全域搜索              ┌─ 模块二：体制内招录 (civil/institution/military/selection)
  t_job_index ────同步──┤
  (source_type)         ├─ 模块三：行业专项 (teacher/healthcare/finance)
                      ├─ 模块四：基层社区 (grassroots/community/public_welfare)

内容独立              ┌─ 模块六：内容管理 (exam_guide + notice)
                      不进入 t_job_index，独立展示

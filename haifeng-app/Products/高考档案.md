# 高考档案模块接口文档

> 基础路径：`/api/v1/app/gaokao`
> 
> 所有接口需要登录（`@RequireLogin`）

---

## 1. 获取改革模式及可选科目

根据省份和年份，获取该省适用的高考改革模式（3+3 / 3+1+2 / 传统文理）及对应的可选科目。

### 请求

```
GET /api/v1/app/gaokao/reform-model
```

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| province | String | 是 | 省份名称，如 "广东"、"浙江" |
| year | Integer | 是 | 高考年份，范围 2020-2030 |

### 响应示例

**3+1+2 模式（如广东 2024）：**
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "reformModel": "3+1+2",
    "subjects": {
      "first": ["物理", "历史"],
      "second": ["化学", "生物", "政治", "地理"]
    }
  }
}
```

**3+3 模式（如浙江 2024）：**
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "reformModel": "3+3",
    "subjects": {
      "first": ["物理", "化学", "生物", "政治", "历史", "地理"]
    }
  }
}
```

**传统文理模式：**
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "reformModel": "传统文理",
    "subjects": {
      "first": ["文科", "理科"]
    }
  }
}
```

### 响应字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| reformModel | String | 改革模式：`3+3` / `3+1+2` / `传统文理` |
| subjects | Map | 可选科目，key 为 `"first"` 或 `"second"`，value 为科目列表 |

## 2. 查询位次

根据省份、年份、科类和分数，查询对应的一分一段位次。

### 请求

```
GET /api/v1/app/gaokao/rank
```

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| province | String | 是 | 省份名称 |
| year | Integer | 是 | 高考年份，范围 2020-2030 |
| subjectType | String | 是 | 科类：`物理类` / `历史类` / `理科` / `文科` |
| score | Integer | 是 | 分数，范围 0-750 |

### 响应示例

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "rank": 12345,
    "sameScoreCount": 50
  }
}
```

### 响应字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| rank | Integer | 位次（累计人数） |
| sameScoreCount | Integer | 同分人数 |

> **注意：** 如果查询的分数在数据库中不存在，返回 `data: null`

## 3. 获取批次分数线列表

根据省份、年份和科类，获取批次分数线。如果当年无数据，自动返回最近5年的最新数据。

### 请求

```
GET /api/v1/app/gaokao/batch-lines
```

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| province | String | 是 | 省份名称 |
| year | Integer | 是 | 高考年份，范围 2020-2030 |
| subjectType | String | 是 | 科类：`物理类` / `历史类` / `理科` / `文科` |

### 响应示例

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "dataYear": 2024,
    "isCurrentYear": true,
    "batches": [
      {
        "batch": "本科批",
        "scoreLine": 439,
        "rankLine": 150000
      },
      {
        "batch": "专科批",
        "scoreLine": 180,
        "rankLine": 350000
      }
    ]
  }
}
```

### 响应字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| dataYear | Integer | 数据来源年份 |
| isCurrentYear | Boolean | 是否为当年数据 |
| batches | Array | 批次列表，按分数线升序排列 |
| batches[].batch | String | 批次名称，如 "本科批"、"专科批" |
| batches[].scoreLine | Integer | 分数线（省控线） |
| batches[].rankLine | Integer | 位次线 |

> **注意：** 如果查询年份及前5年均无数据，返回 `batches: []`

## 4. 保存高考档案

保存当前用户的高考档案（新增或更新）。每个用户只能有一条档案记录。

### 请求

```
POST /api/v1/app/gaokao/archive
Content-Type: application/json
```

### 请求体

#### 必填字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| gaokaoYear | Short | 是 | 高考年份，范围 2020-2030 |
| gaokaoProvince | String | 是 | 高考省份，最多30字符 |
| score | Integer | 是 | 高考总分，范围 0-750 |
| rank | Integer | 是 | 位次，必须大于0 |
| subjectType | String | 是 | 第一科目（见下方科目规则） |
| batch | String | 是 | 所在批次，最多50字符 |
| batchDataYear | Short | 是 | 批次数据年份，范围 2020-2030 |
| batchLineScore | Integer | 是 | 批次省控线，范围 0-750 |

#### 可选字段：选科

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| secondSubjectType | String | 否 | 第二科目（见下方科目规则） |
| thirdSubjectType | String | 否 | 第三科目（见下方科目规则） |

#### 可选字段：各科成绩

| 字段名 | 类型 | 范围 | 说明 |
|--------|------|------|------|
| scoreChinese | Integer | 0-150 | 语文成绩 |
| scoreMath | Integer | 0-150 | 数学成绩 |
| scoreEnglish | Integer | 0-150 | 外语成绩 |
| scoreSubject1 | Integer | 0-100 | 第一科目分数 |
| scoreSubject2 | Integer | 0-100 | 第二科目分数 |
| scoreSubject3 | Integer | 0-100 | 第三科目分数 |

#### 可选字段：外语语种

| 字段名 | 类型 | 说明 |
|--------|------|------|
| foreignLanguage | String | 外语语种，最多20字符 |

#### 可选字段：身体条件

| 字段名 | 类型 | 说明 |
|--------|------|------|
| isColorBlind | Boolean | 是否色盲 |
| isColorWeak | Boolean | 是否色弱 |
| visionLeft | BigDecimal | 左眼视力，范围 0.0-5.5 |
| visionRight | BigDecimal | 右眼视力，范围 0.0-5.5 |
| hasSmellDisorder | Boolean | 是否嗅觉迟钝 |
| heightCm | Integer | 身高（厘米），范围 100-250 |
| weightKg | BigDecimal | 体重（公斤），范围 20.0-200.0 |
| isLeftHanded | Boolean | 是否左利手 |
| hasTattoo | Boolean | 是否有纹身 |
| hasScar | Boolean | 是否有面部疤痕 |
| hasStutter | Boolean | 是否口吃 |

#### 可选字段：身份条件

| 字段名 | 类型 | 说明 |
|--------|------|------|
| isFreshGraduate | Boolean | 是否应届生 |
| politicalStatus | String | 政治面貌，最多20字符 |
| householdType | String | 户籍类型，最多20字符 |
| isPovertyCounty | Boolean | 是否贫困县户籍 |

### 请求体示例

```json
{
  "gaokaoYear": 2024,
  "gaokaoProvince": "广东",
  "score": 580,
  "rank": 12345,
  "subjectType": "物理",
  "secondSubjectType": "化学",
  "thirdSubjectType": "生物",
  "batch": "本科批",
  "batchDataYear": 2024,
  "batchLineScore": 439,
  "scoreChinese": 120,
  "scoreMath": 130,
  "scoreEnglish": 135,
  "scoreSubject1": 90,
  "scoreSubject2": 95,
  "scoreSubject3": 110,
  "foreignLanguage": "英语",
  "isColorBlind": false,
  "isColorWeak": false,
  "visionLeft": 5.0,
  "visionRight": 4.8,
  "heightCm": 175,
  "weightKg": 65.0,
  "isFreshGraduate": true,
  "politicalStatus": "共青团员",
  "householdType": "城镇"
}
```

### 响应示例

```json
{
  "code": 200,
  "msg": "success",
  "data": 1234567890123456789
}
```

> 返回值为档案 ID

## 科目填写规则

**重要：** 科目必须根据改革模式填写，否则会返回 400 错误。

### 传统文理模式

| 字段 | 可选值 |
|------|--------|
| subjectType | `文科` 或 `理科` |

提交后系统会自动映射：
- 文科 → 政治、历史、地理
- 理科 → 物理、化学、生物

### 3+1+2 模式

| 字段 | 可选值 |
|------|--------|
| subjectType（首选） | `物理` 或 `历史` |
| secondSubjectType（再选） | `化学`、`生物`、`政治`、`地理` |
| thirdSubjectType（再选） | `化学`、`生物`、`政治`、`地理` |

> 首选科目只能选1个，再选科目从4个中选2个

### 3+3 模式

| 字段 | 可选值 |
|------|--------|
| subjectType | `物理`、`化学`、`生物`、`政治`、`历史`、`地理` |
| secondSubjectType | `物理`、`化学`、`生物`、`政治`、`历史`、`地理` |
| thirdSubjectType | `物理`、`化学`、`生物`、`政治`、`历史`、`地理` |

> 从6个科目中任选3个

## 注意事项

### 1. 档案保存逻辑

- 每个用户只能有**一条**高考档案
- 首次保存为**新增**，再次保存为**更新**
- 更新时会覆盖所有字段（包括可选字段）
- **请确保前端传递完整数据**，未传递的可选字段会被置为 `null`

### 2. 改革模式自动判断

- `reformModel` 字段**不需要前端传递**
- 系统根据省份和年份自动从 `t_province_reform` 表查询
- 如果省份无配置，默认返回 `传统文理`

### 3. 线差自动计算

- `scoreAboveLine` = `score` - `batchLineScore`
- 系统根据提交的分数和批次省控线自动计算

### 4. 批次数据降级

- 如果查询年份无批次数据，系统会自动查询最近5年的数据
- 响应中 `isCurrentYear: false` 表示使用了历史数据
- `dataYear` 表示实际使用的数据年份

### 5. 错误码

| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 参数错误（如科目不合法、分数超范围等） |
| 401 | 未登录 |

### 6. 科目校验

保存档案时，系统会校验科目是否符合改革模式：
- 传统文理模式：只能填 "文科" 或 "理科"
- 3+1+2 模式：首选只能是 "物理" 或 "历史"，再选只能是 "化学/生物/政治/地理"
- 3+3 模式：只能填 "物理/化学/生物/政治/历史/地理"

### 7. 推荐调用流程

```
1. 调用 GET /reform-model 获取改革模式和可选科目
2. 用户选择科目和填写分数
3. 调用 GET /rank 查询位次（可选）
4. 调用 GET /batch-lines 获取批次分数线
5. 用户填写批次信息
6. 调用 POST /archive 保存档案
```

### XLSX注意事项：

1. 一个是主表，其他的都是外键，有一对多的，有1对1的，1对1的不能使得外键或者能代替外键的重复，
2. 有外键的xlsx都不会传外键id，而是能代替外键唯一的字段，比如大学名，城市名，需要联表找出id并传入（先检查name对应的外键是否存在）
3. 关于如何识别哪一列跟数据库相对的问题，我认为处理方案是按名称，使用 EasyExcel 的 @ExcelProperty("院校名称") 注解即可，针对这次的名称，结合注释等信息自己决定就可以，后续列出来即可。
4. text[]字段的值必须用逗号格式，支持中英文逗号分隔，意思是逗号分隔，代表是不同的值
````
可以写一个自定义的 Converter：

Java
// 伪代码示例
public class StringToArrayConverter implements Converter<String[]> {
@Override
public String[] convertToJavaData(ReadCellData<?> cellData, ...) {
String val = cellData.getStringValue();
if (val == null) return null;
// 用正则同时兼容中英文逗号
return val.split("[,，]");
}
}
然后在 DTO 字段上加上 @ExcelProperty(converter = StringToArrayConverter.class) 即可。
````
5. 可能涉及到类型转换的问题：基础类型（如 Excel 里的 "2026-05-01" 转 Date，"1000" 转 Integer），EasyExcel 和 Spring Boot 默认就能完美转换。
6. 上传过程中遇到了错误，报第几行的具体错误，比如类型错误呀什么的
7. 使用事务保证一致性
8. 如果内容遇到了空值就默认是null，让数据库判断去吧。意思就是除了数据库指定不能为空之外，其他都可以为空
9. 正确顺序：传入的数据表，根据接口不同，有的是主表，有的不是主表
- 主表：检查表头规范性（比如检查看是否有超过规定的表头），检查unique字段的这一列是否有重复的,正确的话就传入数据
- 外键表：先检查院校名（1对1的不能重复，外键的必须在主表里面有对应的）不能重复检查外键表是否有数据，没有数据就报错，有数据就继续
- 错误处理：明确说清楚是什么报错了，比如哪一行，哪个表头，要明确信息
- 最佳实践：这就需要用到 Spring 的事务一致性。
````
在解析 Excel 的过程中，如果有校验不通过的（比如“第5行：院校名称已存在”），把这些错误信息收集到一个 List<String> errorMsgs 里。
如果 errorMsgs 不为空，直接抛出一个自定义异常 throw new BusinessException(String.join(";", errorMsgs))，并中断解析。
只要你的 Service 方法上加了 @Transactional(rollbackFor = Exception.class)，抛出异常后，前面插入的几行数据会自动全部回滚。
````

## 任务2【完成权限管理】
1. 只有管理员才可以看到后台管理，点击之后呈现左边多个父子模块，点击进入模块之后，呈现该模块的页面
2. 有一个权限管理，里面有：角色列表、模块列表、管理员列表
3. 点击模块管理，呈现列表，展示路由与路由的关系，支持增删改,，支持模糊查询module_code，删除软删除 
4. 修改，增加支持的字段是：
module_name   VARCHAR(50) NOT NULL,          
module_code   VARCHAR(50) NOT NULL UNIQUE,   -- 全局唯一
parent_id     BIGINT REFERENCES sys_module(id) ON DELETE CASCADE,
path          VARCHAR(200),                  -- 前端路由 /admin/system/setting
icon          VARCHAR(50),
sort_order    INTEGER DEFAULT 0,
level         SMALLINT NOT NULL,             -- 1=父模块 2=子模块
description   VARCHAR(255),
5. 点击角色模块，呈现列表，展示角色信息，支持增删改,，支持模糊查询role_name，与模块多对多关联
6. 新增或修改角色，除了哪些字段之外，还得有一个关联模块的按钮，点击之后，弹出一个模态框，里面显示所有的模块名称（module_name），可以多选，点击确定之后，将选中的模块code保存到角色模块表中
7. 模块支持父子模块，如果是父子路由，代表所有的子模块都要被赋予角色权限，如果子路由，代表只有子路由被赋予权限
   方案A（存储层）：绑父模块时，把所有子模块也INSERT进 sys_role_module
   方案B（查询层）：只存父模块ID，查询时动态展开子模块
**示例：**
   列表页：树形展示（父模块下面展开子模块）
   返回格式：
   [
   {
   "id": 1,
   "moduleName": "系统管理",
   "moduleCode": "system",
   "level": 1,
   "children": [
   {"id": 101, "moduleName": "控制面板", "moduleCode": "system_dashboard"},
   {"id": 102, "moduleName": "系统设置", "moduleCode": "system_setting"}
   ]
   }
   ]
以上是示例，先实现功能，数据和路由后续再插入
8. 管理员列表，展示管理员信息+联表展示管理员对应的角色名称，支持增删改,，支持模糊查询username，phone,删除软删除，
9. 新增或修改管理员，除了哪些字段之外，还得有一个关联角色的按钮，点击之后，弹出一个模态框，里面显示所有的角色名称，只能选一个，点击确定之后，将选中的角色保存到管理员角色表中赋予它身份
10. 约束：角色名不可重复，模块名不可重复，模块code不可重复

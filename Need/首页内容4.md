**实现需求：**
1. 管理员线下收费，线上管理员后台修改用户信息(member_orders)，把用户类型变成会员。
2. 修改好之后看用户是否有推荐人，有推荐人获得对应的佣金金额，没有不做处理。
3. 用户变成会员或续费成功会发送通知给用户，然后推荐人获取的金额消息也会通知（t_member_notification）
4. 发送通知的同时修改用户表里面的金额数据，要保证数据一致
5. 用户发现佣金金额够一定数量之后，可以点击提现，点提现之后在管理员的后台会出现一条待办事项（t_referral_commission）
6. 根据用户的微信号我们线下付款，然后手动操作完成提现，发送通知给用户，提现完成。

CREATE TABLE IF NOT EXISTS t_member (
id                          BIGSERIAL PRIMARY KEY,
username                    VARCHAR(50) NOT NULL UNIQUE,
password                    VARCHAR(100) NOT NULL,
avatar                      VARCHAR(500),
phone                       VARCHAR(20) UNIQUE NOT NULL,
invite_code                 VARCHAR(8) UNIQUE,
member_type                 VARCHAR(20) DEFAULT 'normal',
expire_at                   TIMESTAMPTZ,
referrer_id                 BIGINT,
referrer_username           VARCHAR(50),
commission_balance          DECIMAL(10,2) DEFAULT 0.00,
commission_total_earned     DECIMAL(10,2) DEFAULT 0.00,

status                      VARCHAR(20) DEFAULT 'active',
last_login_at               TIMESTAMPTZ,

wechat_id                   VARCHAR(255),                       -- 微信号(AES加密存储)
wechat_id_index             VARCHAR(64),                        -- 微信号盲索引(SHA-256)

created_at                  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
updated_at                  TIMESTAMPTZ NOT NULL DEFAULT NOW(),

CONSTRAINT chk_member_status CHECK (status IN ('active', 'disabled')),

);

CREATE INDEX idx_member_type ON t_member(member_type) WHERE is_deleted = FALSE;

#### 任务5【完成推荐佣金表t_referral_commission】
##### 更新用户表
管理员 在修改或开通了用户的会员之后，或者是续费，就先判断他的推荐人referrer_id是否为空，如果为空，那么不做处理。
如果不为空：
数据库：system_settings里面有：**pro_commission_rate**和**vip_commission_rate**字段
1. 管理员在选择会员的类型的时候，如果是pro，那么对应pro_commission_rate，如果是vip，那么对应vip_commission_rate（vip续费对应他）
2. 根据当前的金额，进行比率计算，算出属于邀请人的佣金金额
3. 找到邀请码对应的用户，通过佣金比例和用户花费的金额计算出实际得到金额，把用户表里面的：**commission_balance**和**commission_total_paid**在原有基础上增加即可
4. 添加一条推荐佣金记录，记录会员ID，被推荐人ID，推荐人姓名和电话，被推荐人姓名和电话，关联订单ID，订单金额，佣金比例，佣金金额

##### 更新数佣金表
**t_referral_commission列表**
1. 展示推荐人姓名和电话，被推荐人姓名和电话，关联订单id，订单金额，佣金比例，佣金金额
2. 模糊查询：推荐人手机号和姓名，被推荐人的手机号和姓名，订单编号
